
JWT (JSON Web Token) — это
компактный, безопасный способ передачи данных между клиентом и сервером (обычно для аутентификации и авторизации), 
который позволяет серверу проверять права пользователя без постоянного обращения к базе данных. JWT экономит ресурсы, 
ускоряет работу API и идеально подходит для микросервисов и мобильных приложений. 
Основные функции и преимущества JWT:

    Аутентификация (вход в систему): После входа пользователя сервер генерирует токен, который клиент отправляет в каждом запросе, 
подтверждая свою личность.
    Авторизация: JWT содержит информацию о правах доступа (ролях) пользователя, позволяя получить доступ к защищенным ресурсам API.
    Безсостоятельность (Stateless): Серверу не нужно хранить сессию в базе данных. Вся необходимая информация о пользователе уже 
зашифрована внутри токена.
    Безопасность: Токены подписываются криптографической подписью, что гарантирует целостность данных — их нельзя подделать без 
секретного ключа.
    Single Sign-On (SSO): Удобны для использования в нескольких сервисах сразу: аутентифицировавшись один раз, 
пользователь получает доступ к разным приложениям. 

Почему используют JWT:

    Снижение нагрузки на БД: Не требуется постоянный запрос к базе для проверки валидности сессии при каждом действии.
    Масштабируемость: Отлично работают в микросервисной архитектуре, где каждый сервис может проверить токен самостоятельно.
    Кросс-доменность: Подходят для передачи данных между разными доменами (например, api.example.com и app.example.com)
    
    
    
    
HS256 (HMAC с SHA-256) — это симметричный алгоритм подписи JSON Web Tokens (JWT), использующий один секретный ключ для создания и проверки подписи. Он обеспечивает целостность данных: если данные изменены, подпись станет недействительной. Поскольку ключ общий, его сложнее безопасно распределять, чем в асимметричном RS256. 
Ключевые особенности HS256:

    Симметричность: Один и тот же секретный ключ используется и авторизационным сервисом, и клиентом.
    SHA-256: В основе лежит криптографическая хеш-функция, создающая подпись длиной 256 бит.
    Область применения: Чаще всего используется в простых архитектурах, где сервер авторизации и потребитель токена — это одна система (или доверенные системы).
    Безопасность: Ключ должен быть длинным и случайным, чтобы избежать атак методом перебора. При компрометации ключа злоумышленник может создавать свои JWT. 

Сравнение с RS256:
В отличие от HS256, алгоритм RS256 является асимметричным, использует закрытый ключ для подписи и открытый ключ для проверки, что безопаснее для распределенных систем.
